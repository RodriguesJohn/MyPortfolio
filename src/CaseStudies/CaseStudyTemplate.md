# Case Study Template

## Project Overview 
- Brief project description (1-2 sentences)
- Key challenge or problem statement
- Goals and objectives

## Hero Shot
- Main visual that captures the essence of the project

## Outcome 
- Key results and metrics (if available)
- Impact on users/business
- Success indicators

## Project Context 
- **My Role** - What you specifically contributed
- **Background** - What led to this project
- **Timeline** - Project duration and key milestones
- **Collaborators** - Team members and their roles
- **Platform** - Web, iOS, Android, etc.
- **Category** - Product type, industry, etc.
- **Tools Used** - Design and collaboration tools
- **Constraints** - Technical, business, or resource limitations

## Design Process 

### Discovery & Research
- User research findings
- Competitive analysis insights
- Problem validation
- User personas or target audience

### Ideation & Strategy
- Key design decisions and rationale
- Information architecture
- User flows and journey mapping
- Design system work (if applicable)

### Design & Iteration
- Before and after comparisons
- Design iterations and evolution
- Prototyping approach
- How research findings led to pivots

### Validation & Testing
- Testing methods (usability, A/B, etc.)
- Feedback incorporation
- Iteration cycles

### Final Deliverables
- Clearly show the work
- Key screens and interactions
- Design system components (if applicable)

## Conclusion & Results
- Final outcomes and metrics
- Key learnings and takeaways
- What worked well / what could be improved
- Next steps or future considerations

## Additional Sections (Optional)
- **Accessibility** - How accessibility was considered
- **Technical Implementation** - Design-to-code handoff notes
- **Reflection** - Personal learnings and growth
